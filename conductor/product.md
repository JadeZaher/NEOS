# NEOS — New Earth Operating System

## What It Is

NEOS is a monorepo housing a governance platform with BFF (Backend For Frontend) architecture. A Python/Sanic API serves a React frontend, delivering AI-powered governance facilitation for non-sovereign communities.

The platform enables groups to coordinate without centralized authority using a stack of 54 governance skills across 10 layers — from agreement creation to exit portability.

## Target Users

1. **Ecosystem Architects** — Designers who adopt NEOS as a blueprint and configure governance for their community
2. **Community Members** — Participants in NEOS-based ecosystems (starting with OmniOne in Bali) who use the platform daily for governance, learning, and coordination
3. **AI Agents** — Claude-powered agents that guide participants through governance processes using the skill stack

## First Ecosystem: OmniOne

Stewarded by Green Earth Vision (GEV), a 501(c)(3) non-profit. Operating in Bali with plans for Costa Rica, Mexico, Brazil. Physical presence via the SHUR (Synergy Hub Universal Residencies) network.

## Key Features

- **Governance Dashboard** — Real-time SSE-powered dashboard with AI chat
- **Course & Team Management** — Learning platform for governance onboarding and assessment
- **54 Governance Skills** — Structured markdown skills an AI agent can execute
- **Agreement Management** — Full lifecycle: create, amend, review, sunset
- **ACT Decision Protocol** — Advice → Consent → Test (not voting)
- **Conflict Resolution** — Harm circles, NVC dialogue, repair agreements
- **Capture Detection** — Monitors for capital, charisma, emergency, and ossification capture

## Success Criteria

1. OmniOne members can use the platform daily for real governance operations
2. The skill stack is forkable — another ecosystem can adopt NEOS core with their own configuration
3. The React frontend and Python API work seamlessly via BFF architecture
4. Deployed on Railway with no Supabase dependency
